# Concept Glossary Starter

This public skill ships with an empty glossary. Do not include a maintainer's personal reading history here.

Use this file only as a local, user-owned glossary after installation. When a user's reading workflow discovers a reusable concept, append a compact entry using this shape:

```markdown
### Concept Name (Optional English Name)
- **Source paper**: [[Paper Note]]
- **Definition**: One concise definition grounded in the paper.
- **Why it matters**: One sentence about why this concept is useful across papers.
- **Related links**: [[Related Concept]]; [[Related Method]]
```

Guidelines:

- Keep entries short; detailed explanation belongs in `02-Concepts`.
- Do not add common background terms unless the paper uses them in a field-specific way.
- Do not publish user-specific concept entries in a shared skill package.
- For public redistribution, keep this file as a starter template.
